# Docs and Decisions Skill

## Goal

Keep workflow and durable context aligned with implementation reality.

## Update rules

- Update `SESSION.md` at the end of every session.
- Update `DECISIONS.md` when a durable decision is made.
- Update `RISKS.md` when risk changes materially.
- Move unanswered cross-session questions into `docs/OPEN_QUESTIONS.md`.
- Record when an unanswered question exposes a missing or contradictory upstream handoff input.

## Constraints

- Do not use narrative summaries in place of structured fields.
- Do not let open questions disappear.
- Keep session notes factual and implementation-specific.
